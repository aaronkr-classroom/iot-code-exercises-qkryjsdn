1.Q1
  - (a) 디바이스의 센서와 액추에이터의 역할을 비교하여 설명하세요.
  -  센서란, 주변 환경의 다양한 물리적 상태를 감지하고 이를 전기 신호로 변환해주는 장치 즉, 정보를 받아들이는 역할이다. 액추에잍는 전기 신호를 받아 물리적인 움직임이나 변화를 만들어내는 장치, 즉 정보를 실행하는 역할을 한다. 따라서 센서는 외부 정보를 입력받는 장치이고, 액추에이터는 그 정보를 바탕으로 동작을 수행하는 출력 장치이다.
  - (b) 디바이스와 대시보드에서 동일한 프로그래밍 언어를 사용하지 않는 이유를 설명하세요.
  - 디바이스는 센서와 액추에이터를 제어하는 임베디드 환경 속에서 동작한다. 이 환경은 메모리, 처리능력이 제한적이기 때문에, C와 CC++같은 언어가 주로 사용된다. 대시보드는 사용자 인터페이스 제공 및 데이터를 시각화하는 역할을 하기에 주로 웹이나 어플리케이션 환경 속에서 동작하고 사용자와의 상호작용이 중요시되기 때문에 JavaScript, Python 등과 같은 언어가 사용된다.
  - (c) 브로커와 같은 정보 전달 전담 서버의 이점을 설명하세요.
  - 브로커를 통해 데이터를 주고 받으면 하나의 시스템이 변경되더라도 다른 시스템에 영향을 최소화 할 수 있기에 결합도 감소 및 시스템 유연성의 이점, 브로커가 메시지를 중간에서 관리하고 있기 때문에 수신 측이 동작하지 않아도 데이터를 저장했다가 재전달할 수 있어 데이터 손실을 줄일 수도 있다. 마지막으로 새로운 디바이스나 서비스가 추가될 때 기존 시스템을 크게 수정하지 않고 브로커에 연결만 하면 되기에, 시스템 확장에도 용이하다는 이점이 있다.
